몆달동안 나는 SingIt 을 완성하고자 UI 디자인을 짜고, 노래의 헤르츠와 유저의 음성인식을 통해 pitch 떨림을 노래 헤르츠와 비교 -> 노래를 잘 불렀는가? O / X 로 판별 기능을 구현하고자 했지만 여러 검색끝에 결국 다른 방식을 택해야 겠다는 결론에 도달했다.

PitchDetector 를 사용할 수 없는 이유는 해당 노래를 부른 가수와 유저간의 부르는 방식, 음의 높낮이 등이 달라서 어렵게 구현한다 해도 제 기능을 다 못할 것 같았다.

그 외에도 검색을 통해 머신러닝과 수 많은 api 를 검색해 보았다.

<br>

머신러닝. TensorFlow 를 이용하는건 전문적인 지식이 필요하며 만들어진 머신러닝을 이용하고자 하여도 내가 원하는 만큼의 퀄리티를 내는 AI 머신은 google 에 hum to search 정도였다.

<br>

API 쪽을 열심히 찾다가 Rapid Api 라는 웹사이트를 발견하게 되였고 희망에 가득찬듯이 하루종일 트라이 해봤다.

하지만 shazam, Musixmatch , ACRCloud , SoundHound 모두 직접 부르며 사용해 보아도 결국 humming (음) 으로는 노래를 못 찾거나 틀렸고, 원곡을 틀어야만 제 기능을 했다.

<br>

내가 만들고자 하는 SingIt 은 여러 사람들이 모여 아는 곡이 나오면 그 곡을 무반주로 부르며 원곡과 비슷하게 부르면 통과하는 방식이다. 고로 순수히 humming 만으로 노래를 판별할 수 있어야 하며, 2달동안 google 의 AI 러닝머신 hum to search 기능 만이 유일해 보였다.

<br>

허나 이 기능은 지금 구글측에서 API 로 풀어둔 상태가 아니며 2020년도에 개발이되어 아직 안드로이드 구글앱에서만 기능을 사용할 수 있으며, 추후 API 로 나올 수 있으나 안 나올수도 있기에 포기해야 하였다.

<br>
<br>

꾸준히 검색하며 새로운 자료를 찾아볼 것이다. 새로운 아이디어가 떠오르면 SingIt 의 플레이 방식을 바꿔볼까 생각중이다.

아무래도 다른 프로젝트를 새로 동시진행 해야할 것 같다.
